HTML 사전(계속 추가)

HTML : HyperText Markup Language. 하이퍼텍스트 마크업 언어. - 비선형적 초월 이동이 가능하며 구조적 의미를 가진 데이터로 이루어지는 문서를 작성하는 언어. - HyperText: 초월. 고전적 문서의 한계를 넘어선. - Markup: 데이터에 이름을 붙인다. 즉, 해당 데이터의 구조적 의미를 설명한다. <!DOCTYPE html> : 문서 종류 - 브라우저 쪽에 해당 문서의 종류를 설명. <html lang="en"> : 문서의 주된 언어. - 스크린 리더의 동작에 영향. 웹 접근성 측면에서 중요함. <head> : 기계가 식별할 수 있는 문서 정보(메타데이터)를 담는다. - 제목, 스크립트, 스타일 시트 등. <title>Document</title> </head> <body> -> 콘텐츠를 담는다. <h1>hello world</h1> </body> </html> <h> : heading. 제목. - 32px - 1~6단계 <p> : paragraph. 문단. <a> : Anchor. 앵커. 초월 닻. - <a href = "URL"> : 다른 문서로 이동할 수 있는 하이퍼링크를 생성. - URL : Uniform Resource Locator. web address. : 주소. 링크. : 네트워크 상에서 자원이 어디 있는지를 알려준다. - <a href = "URL" target = "_blank, _self, _parent, _top..."> : 연결된 문서가 로드될 구역을 지정. - _self : 현재 프레임에서 연결된 문서를 로드(디폴트 값). - _blank : 새로운 탭에 연결된 문서로를 로드. - _parent : 현재 프레임의 부모 프레임에서 문서를 로드. - _top : 전체 화면에 문서를 로드. - 프레임 이름 : 특정한 프레임에서 문서를 로드. - <a href = "URL" download (="File Name")> : 연결된 파일을 다운로드. - 파일 이름을 속성값으로 설정해 내려받아지는 파일 이름을 변경할 수 있음. ./ : 현재 폴더(를 의미). - ex) <a href = "./index.html">click</a> #id : 페이지 내 해당 id를 가진 태그로 화면을 이동시킨다. - ex) <a href = "#move">click</a> <strong> : 굵게 강조. 태그 안의 내용을 bold하게 표시. - 스크린 리더가 강조해서 읽음. <b> : 굵게 강조. 태그 안의 내용을 bold하게 표시. - 스크린 리더가 강조하지 않음. <em> : 기울임 강조. 태그 안의 내용을 italic하게 표시. - 스크린 리더가 강조해서 읽음. <i> : 기울임 강조. 태그 안의 내용을 italic하게 표시. - 스크린 리더가 강조하지 않음. <mark> : 색 강조. 형광펜처럼 표시. <hr> : 가로선 <abbr> : abbreviation. 약어. 준말 표시. - ex) <abbr>웅짱</abbr>(웅비는 짱이다) <kbd> : 키보드 키 표시. <pre> : 있는 그대로 표시. <code> : 코드 블록 표시. <cite> : 인용 표시. Block. <q> : 인용 표시. Inline. <ol> : 오더 리스트. 순서가 있는 목록을 생성한다. - 리스트 내용 앞에 순서를 나타내는 기호가 생성됨. - type 지정 가능. <ol type = "A(, a, I, i, ...)"> <ul> : 언오더 리스트. 순서가 없는 목록을 생성한다. - 리스트 내용 앞에 점이 생성됨. <li> : list. 목록의 내용. <dl> : definition list. 정의 목록. <dt> : definition term. 정의 용어. <dd> : definition description. 정의 설명. - ex) 사업자 번호(dt): 번호(dd), 대표 이름(dt): 이름(dd) - 주로 홈페이지 하단 footer 구역에 들어간다. <div>, <span> : 구역을 지정. - 시멘틱한 HTML 구성을 지향해야 하기 때문에, - 해당 데이터를 꾸미는 목적의 태그가 따로 없을 경우에만 사용하기. - 태그는 용도에 맞게 쓰자. <img> - 이미지 삽입. - ex) <img src = "URL(./src/a.jpg)" scrset = "" alt = ""> - src = "URL" : source의 약자. 소스. : 속성값으로 URL을 가진다. - alt : alternative. 대체 텍스트. : 속성에는 이미지에 대한 설명을 적는다. 스크린 리더가 읽는다. - scrset : 이미지 소스의 세트. 같은 비율의 다양한 크기를 가지는 동일 이미지들. - 내용 추가해야 함. 구성 header. nav. / section.(header. / article. / footer.) / aside footer article : 아티클. 완전히 독립적으로 역할할 수 있는 요소. - 포스트, 기사 등. section : 섹션. 아티클을 감싸주는 용도. <input> : 특정 요소를 삽입한다. - ex) <input type = "" name = "" id = ""> - type 속성. 속성값으로는 text, password, date, time, range, color, radio, checkbox... - name 속성. name = 입력값 형식으로 전송됨. - readonly 속성. 태그를 읽기 전용으로 만듦. -> ??? - maxlength 속성. 최대 글자 수를 지정. - required 속성. 해당 태그를 필수 태그로 만듦. - autofocus 속성. 페이지 로딩과 동시에 이 속성을 지정한 태그로 포커스 이동. - placeholder 속성. 태그에 입력할 값에 대한 힌트를 표시. - pattern: 정규표현식을 사용하여 특정 범위 내의 유효한 값을 입력받음. -> ??? <label> : 라벨. input 태그 앞에 사용되어 input 요소에 이름을 붙여준다. - ex) <label for = "id">아이디</label> - for 속성을 사용할 수 있음. 클릭 시 같은 속성 값(id 값)을 가진 input 요소로 초점을 이동시킴. <form> : 입력받은 데이터를 정리해 한꺼번에 서버로 전송한다. - ex) <form action = "http:// ... " accept-charst="utf-8" name = "person_info" method = "get"> - input & label 요소를 포함함. - name = value 형식으로 정리됨. - ex) l?아이디=zo0ozzz&패스워드=123&성별=남&언어=Python#move - action 속성. : 데이터를 전송할 서버 쪽 스크립트 파일을 지정. - 속성값이 없으면 자기 자신에게 값을 전달함. - name 속성. : 폼을 식별하기 위한 이름을 지정. - accept-charset 속성. : 폼 전송에 사용할 문자 인코딩 지정. - method 속성. : 폼을 서버에 전송할 http 메소드를 지정(get or post). - form을 구성하는 요소들. - <field> : 연관된 요소들을 그룹화. - <legend> : <field>의 하위 태그로 요소 그룹의 이름을 지정. - <select> : 항목을 선택할 수 있는 태그. - size 속성. : 한번에 표시할 항목 수. - multiple 속성. : 다중 선택 허용. - <optgroup> : 태그 안에서 목록들을 그룹화. - label 속성을 사용해 그룹 이름을 지정. - <option> : 목록을 나타냄. -
