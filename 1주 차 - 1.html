HTTP 프로토콜
: Hypertext Transfer Protocol. 상호 간에 정의한 규칙. 데이터를 주고 받기 위해 정해졌다.

- 특징
: 상태가 없다. stateless.
: 데이터를 주고 받기 위한 각각의 데이터 요청이 서로 독립적으로 관리된다.
: 즉, 이전 데이터 요청과 다음 데이터 요청이 서로 관련이 없다.
    -> 서버는 별도의 추가 정보를 관리하지 않아도 된다. 
    -> 다수의 요청 처리 및 서버의 부하를 줄일 수 있다.
    -> 서버와 클라이언트가 상태를 주고받을 때 스토리지와 쿠키가 필요하다.

02. HTML 주요 태그

HTML
: 눈에 보이는 정보를 표시한다.
    - 텍스트 정보, (정보성을 띄고 있는)이미지 등.

    - 예) header, nav, main, article, footer 등, 정보가 들어가는 영역.

HTML 태그 구성

- 태그
: 태그명. 고유의 기능이 있다. 예) 제목 h1-h6, 이미지 img

- 속성
: class id 등 이름. 디자인적 작업.

- 속성값
: 속성을 바탕으로 어떠한 기능을 적용시킬지. 구체적인 명령을 입력한다.

- 컨텐츠
: 정보

- 닫힌 태그

HTML 문서의 기본 구조

1. <!DOCTYPE html>
: 문서에 사용될 언어를 선언.
: HTML5라는 언어를 사용할 것.

2. <html>...</html>
: HTML 문서의 시작과 끝을 의미.
 - 모든 HTML 태그들은 <html> 태그 안쪽에 입력해야 한다.

3. <head>...</head>
: 웹사이트의 간단한 요약 정보를 담는 영역.
 - 웹사이트에서 노출되지 않는다.

4. <body>...<body>

: 모든 문자를 웹 브라우저에서 깨짐 없이 표시하겠다는 의미.

웹사이트 내용

<img>

- 상하 배치 작업 불가능.

04 레이아웃에 영향을 미치는 속성

1. display
: blcok inline 요소의 성격을 바꿀 때 사용함.
- inline-block을 사용하면 두 요소의 성격을 모두 가짐
-
display: inline
display: block
display: inline-block;
: x축 정렬이면서 공간에 대한 크기를 만들고 상하 배치 작업도 가능

2. float
: 어떤 오브젝트를 왼쪽/오른쪽에서부터 정렬시키고자 할 때 사용
: 공중에 띄우다 레이어층을 만듦.
- float: left;
- float: right;
레이어가 겹쳐지지 않는 상태로 왼쪽에서부터 정렬시키고 싶은 경우
-> 연속적으로 입력
.left{float: left;}
.next{float: left;}

부모 태그의 크기를 기준으로 왼쪽 끝 오른쪽 끝이 설정된다.

플롯이 적용된 영역의 크기보다 부모의 위드값이 작다면?
-> 레이아웃이 틀어진다. 동일하거나 작아야 한다.


3. clear
: 플롯을 사용했다면 항상 따라오는 속성
: 레이어층이 겹치는 형태로 화면에 출력이 되기 때문에
: 플롯 기능을 꺼준다.
플롯을 마지막으로 사용한 지점 그 다음에 나오는 태그에 clear를 넣어준다.
footer{clear: both}
: 양쪽 모두를 꺼주겠다는 의미
