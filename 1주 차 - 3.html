오늘은 CSS transform, transition, ainmation, media 속성에 대한 강의가 진행됐다.

* transform

transform
: 웹 사이트의 특정 영역에서 오브젝트의 형태, 위치를 조정하는 속성.

- 속성값

  - rotate(45deg)
  : 회전하다(시키다). 대상을 입력한 각도만큼 평면적으로 회전시킨다.
  : 대상을 45도만큼 회전시킨다.
    - deg(degree) 단위 사용.
    - 음수도 입력 가능.

  - scale(2, 3)
  : 규모. 대상의 크기를 확대/축소시킨다.
  : 대상의 width를 2배, height를 3배로 조정한다.
    - scale(x(= width의 x배로), y(= height의 y배로)).
    - 대상을 축소시키려면 0과 1사이의 숫자를 입력함.

  - skew(10deg, 20deg)
  : 비스듬히 움직이다. 입체적으로 각도를 비틂.
  : 대상을 x축으로 10도, y축으로 20도 튼다.
  -> ?? 입체적으로가 뭘까..
    - deg(degree) 단위 사용.

  - translate(100px, 200px)
  : 대상의 위치를 변경한다.
  : 대상을 x축으로 100px, y축으로 200px만큼 옮긴다.
    - 이동의 기본적인 기준점은 대상의 중심점.
    - 마진, 패딩을 이용해 위치를 변경하는 것과 비슷한 효과를 낼 수 있음. 최근엔 이 속성을 많이 사용한다고.

- prefix(지금은 사용하지 않음)
: 다른 버전의 브라우저에서 이 속성을 작동하게 하기 위해 넣어주는 문구.
  - transform은 CSS3에 추가된 최신 속성인 관계로 모든 브라우저에서 지원하지는 않기 때문. 브라우저 입장에서도 처음 보는 상대에게 경계심을 푸는 데까지는 시간이 좀 필요한 법이다.
  - prefix를 달아주면 각 브라우저의 하위 버젼에도 정상적으로 작동시킬 수 있음.
- 예)
.transition{
  -webkit-transform: translate(100px, 200px);
  -moz-transform: translate(100px, 200px);
  -ms-transform: translate(100px, 200px);
  -o-transform: translate(100px, 200px);
}
- 종류
  -webkit-
    : 크롬, 사파리
  -moz-
    : 파이어폭스
  -ms-
    : 익스플로러 9.0 이상
  -o-
    : 오페라

* transition

transition
: 대상의 변화 과정을 지정한 시간에 걸쳐 보여주는 속성.

- 종류

  - transition-property: width;
    - 속성값
      : 변화시키고자하는 css 속성.
      - all
      : 대부분의 속성값들에 영향을 주겠다.

  - transition-duration: 2s;
  : 변화에 소요될 시간.

  - transition-timing-function: linear;
  : 변화 과정에서의 속도를 규정.
    - 속성값
      - linear
      : 일정하게

  - transition-delay: 1s;
  : 변화의 조건이 갖춰짐과 변화의 시작 사이의 시간을 지정.

- 가상 선택자
: 변화의 조건을 규정.
  - CSS에 이미 지정되어 있다.

  - 종류

      - :hover
      : 마우스를 올렸을 때라는 조건.
        - 예)
        .transition: hover{width: 300px;}
        : 마우스를 올렸을 때 가로폭을 300px로 변화시킨다.

- 축약형
: 여러 transition 관련 속성들의 속성값을 한 줄에 선언하는 방식.
  - 프로퍼티, 듀레이션, 타이밍 펑션, 딜레이 순.
  - 근데 순서 상관없이 입력해도 됨.
  - 주의할 건 duration 속성값과 delay 속성값에는 순서가 있다는 것. 앞 순서로 나오는 숫자가 duration의 속성값으로 설정됨.
    -> 숫자가 하나인 경우에는 duration의 속성값으로 처리됨.

  - 예)
    .transition{transition: width 2s linerar 1s;}
    .transition: hover{width: 300px}
    : 대상에 커서를 올리고 1초 후에 위드값을 2초 동안 일정하게 변화시키는 트랜지션을 적용시키겠다.

* animation

animation
: 별도의 조건 없이 페이지가 로드되자마자 대상을 변화시키는 속성.
  - 특징은 변화의 트리거가 되는 특별한 조건을 갖지 않는다는 것. 이런 특징이 transition 속성과의 가장 큰 차이점.

- 종류

  - animation-name: changeWidth;
  : 애니메이션의 이름.
    - 임의로 작성 가능.

  - animation-duration: 3s;
  : 애니메이션이 동작하는 시간

  - animation-timing-function: linear;
  : 애니메이션이 동작하는 속도.

  - animation-delay: 1s;
  : 페이지가 로드되고 애니메이션이 시작될 때까지의 시간.

  - animation-iteration-count: 6;
  : 애니메이션 반복 횟수.

  - animation-direction: alternate;
  : 애니메이션 진행 방향.
    - alternate: from -> to -> from
    : 시작점에서 끝점으로 갔다가 다시 시작점으로 돌아오는 동작.
    - normal: from -> to, from -> to
    : 시작점에서 끝점으로 진행되는 동작.
      - 각 동작당 1회로 처리된다.
    - reverse: to -> from, to -> from
    : 끝점에서 시작점으로 진행되는 동작.
      - 각 동작당 1회로 처리된다.

- @keyframes
: 작동시킬 애니메이션을 선택하고 동작의 시작점과 끝점을 지정한다.
- 중괄호 안쪽에 from(시작점)과 to(끝점)을 지정.
- @keyframes 옆에 작동시킬 애니매이션을 해당 애니매이션의 이름으로 선택한다.

- 예)
@keyframes changeWidth{
  from{width: 300px;}
  to{width: 600px;}
}
: width 300px에서 시작해서 width 600px까지 도달한다.

- 축약형
: 네임, 듀레이션, 타이밍 펑션, 딜레이, 반복, 진행 방향.
  - transition과 마찬가지로 duration과 delay를 제외한 순서는 상관없음.
  - 앞에 나오는 숫자가 듀레이션이라는 점을 유의.

* animation 응용

transform & animation
: 애니메이션의 시작점과 끝점으로 트랜지션 속성을 준다.

- 예)
.box1{
  animation: rotation 1500ms linear infinite alternate;
}
@keyframes rotation{
  from{transform: rotate(-10deg);}
  to{transform: rotate(10deg);}
}
  - 1500ms = 1.5초

- prefix 작성 시 유의 사항(지금은 사용되지 않음)
: 애니메이션에 프리픽스를 달았다면 키프레임즈, 키프레임즈 안쪽에도 프리픽스를 달아줘야 한다.

- 예)
.box1{
  -webkit-animation: rotation 1500ms linear infinite alternate;
}
@-webkit-keyframes rotation{
  from{-webkit-transform: rotate(-10deg);}
  to{-webkit-transform: rotate(10deg);}
}

* 미디어쿼리

미디어쿼리
: 다양한 화면 크기에 대응하는 반응형, 적응형 웹사이트를 건설하는 데 쓰이는 CSS 기법.
: 화면 크기의 변화에 따라 최적화된 결과값을 보여주기 위한 방법.

- 예)
@media (min-width: 320px) and (max-width: 800px){
  .media{
    width: 300px;
    height: 300px;
    background-color: yellow;
  }
}
: 브라우저의 가로폭이 최소 320px 최대 800px이 되었을 경우, 중괄호 안의 CSS 속성을 적용하겠다는 의미.

- 대응 방식

  - 반응형
  : 화면 폭이 변할 때 웹사이트 요소들의 자연스러운 리사이즈 효과가 일어나게 하는 방식.

  - 적응형
  : 화면 폭이 변할 때 뚝뚝 끊기며 레이아웃이나 폰트 크기가 달라지게 하는 방식.

  -> ??? 솔직히 이해도 구분도 안 된다. 따로 조사해서 설명 수정해야 할 듯.

- min-width:/max-width:로 브라우저의 가장 짧은/가장 긴 가로폭을 설정함.
- @media 안쪽에는 CSS 선택자로 대상을 특정하고 변화시킬 속성과 변화의 목적지가 되는 속성값을 설정함.

- 미디어쿼리가 정상적으로 출력되는지 확인하는 방법
: 개발자 도구 -> 모바일 화면으로 전환

- 미디어쿼리 사용 시 주의 사항
  - viewport 설정
  : <meta name="viewport" content="width=device-width, initial-scale=1.0">
    - 미디어쿼리를 사용할 때는 viewport라는 메타 태그의 속성값을 반드시 기입해야 한다.
    - 모바일이나 태블릿은 미리 설정되어 있는 viewport로 인해 미디어쿼리가 제대로 작동하지 않을 수 있음.
    - 따라서 미디어쿼리를 사용할 때는 <meta> 태그의 속성 중 하나인 viewport를 입력해 너비와 배율을 조정해야 함.
    - 이 코드를 입력하지 않으면 비율이 깨진 형태로 웹사이트가 출력될 수 있음.

  - 속성과 속성값

    - name="viewport"
    : 기기 화면에 표시되는 영역을 의미. 너비와 배율을 설정할 때 사용하는 메타 태그의 속성 중 하나.

    - content="width=device-width
    : 뷰포트의 width 값을 기기 화면의 width 값으로 대체하겠다.

    - initial-scale="1.0"
    : 모바일이나 태블릿, 노트북 등으로 접속했을 때 초기 비율을 항상 1.0으로 한다.

- 미디어쿼리 적용 시 CSS 속성 상속
: 바깥쪽에 지정되어 있는 속성에 대하여 미디어쿼리 안에서 해당 속성을 따로 지정해주지 않으면 바깥쪽에 있는 CSS 속성을 상속받는다.
  - 미디어쿼리로 웹페이지의 모바일 버전을 설정값을 만들 때는 pc 버전에 어떤 CSS 속성을 사용했는지 반드시 확인해야 한다.
  - 해당 속성을 상속받지 않으려면 미디어쿼리 안의 속성값을 none;으로 설정한다.

- 예)
.media{
  width: 500px;
  height: 500px;
  background-color: yellow;
}

@media(min-width: 320px) and(max-width: 800px){
  .media{
    width: 300px;
    height: 300px;
    background-color: none;
  }
}
