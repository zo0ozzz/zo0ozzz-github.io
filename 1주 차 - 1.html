* 웹을 구성하는 요소

웹 사이트 구성 요소와 역할

- HTML
:웹 사이트가 제공하는 정보(데이터) 그 자체이자, 그 데이터를 구조화하는 도구.
: 구조화란, 각가의 요소들을 특정한 방식으로 결합시켜 그것을 그저 요소들의 모임이 아니라 명확한 목적을 가진 형태의 새로운 무엇으로 만드는 것.
  - HTML은 그 자체로 데이터임과 동시에 그 데이터를 구조적 의미를 가진 header 태그나 main 태그 등으로 묶음으로써 데이터 무더기를 웹 사이트라는 형태로 구조화함.
  - 정보는 텍스트 정보, (정보성을 띄고 있는) 이미지 등.
  - 정보들의 특징을 파악하여 특징별로 구역을 설정하는 태그들은 header, nav, main, article, footer 등.

- CSS
: HTML에 디자인을 부여하는 도구.
  - 데이터와 데이터들의 구조를 웹 사이트 건설자의 의도에 맞게 시각적으로 꾸밈.

- JS
: 기본적으로 정적인 HTML 요소에 사용자의 행위를 입력받고 그에 대한 반응을 출력하는 기능을 부여하는 도구.
  - 예) 사용자가 카테고리 버튼을 누르면 해당 카테고리에 속해 있는 콘텐츠들이 화면에 출력되게 함.

웹 사이트 제작 시 고려해야 할 사항

1. 웹 표준
: 웹사이트를 작성할 때 따라야 하는 규칙.
  - 강제성은 없지만, 웹 표준을 지켜 건설된 웹 사이트의 데이터는 검색 엔진의 파싱 기준을 충족시켜 검색 엔진에 더 잘 걸리게 됨.

2. 웹 접근성
: 장애가 있어도 웹 사이트를 이용할 수 있게 하는 방법.
  - 색맹, 시각 장애인 등이 웹 사이트를 이용하기 힘들게 하는 장애물들을 제거하거나 웹 사이트를 이용하는 다른 방법을 제공함.

3. 크로스 브라우징
: 크롬, 사파리, 익스플로러 등 여러 브라우저에서 정상적으로 출력되게 제작한다.

HTML 문서의 기본 구조를 잡는 태그

1. <!DOCTYPE html>
: 문서를 작성하는 데 사용될 언어를 선언하는 구역.
: 이 문서는 html5로 작성된 문서다, 라고 컴퓨터에게 알려준다.

2. <html>...</html>
: HTML 문서의 시작과 끝을 의미.
  - 모든 HTML 태그들은 <html> 태그 안쪽에 입력해야 한다.

3. <head>...</head>
: 웹 사이트의 요약 정보를 담는 구역.
  - 웹 사이트 자체에는 노출되지 않는다.

4. <body>...<body>
: 웹 사이트에서 출력되는 정보를 담는 구역.
  - 시각적으로 출력되는 것은 모두 여기 담긴다고 생각하면 편함.

5. <meta charset="UTF-8">
: character setting의 약자.
: 모든 문자를 깨짐 없이 출력하라, 라고 컴퓨터에게 알려준다,
  - 넣지 않으면 한글 등 특정 문자가 제대로 출력되지 않을 수 있음.
    -> ??? 넣지 않으면 왜 깨질까?

6. <title>...</title>
: 브라우저의 탭에 표시되는 웹 사이트의 이름을 지정하는 구역.

주요 태그

<img>
: 이미지를 삽입하는 태그.

- 예) <img src="logo.png(이미지 파일의 경로)" alt="회사 로고">

- 속성
  - src="url"
  : 이미지의 파일 경로.
  - alt="회사 로고"
  : 웹사이트에 이미지가 출력되지 못했을 때 이미지를 텍스트 정보로 대체할 수 있다.
    - 이미지가 로드되지 못했을 때 빈 공간만 나오면 안 됨. 텍스트 정보로 이미지가 들어가야 했을 자리를 표기해야 함.
    - 이미지에 대한 정보를 적으면 텍스트 리더를 사용하는 사용자가 이미지를 보지 않고도 이미지에 대해 파악할 수 있음.
  - width/height
  : 이미지의 사이즈를 지정한다.
    - width를 조정하면 원래 이미지의 비율에 맞춰 높이가 자동으로 변경됨.

<h>
: heading. 제목이나 부제목을 표현한다.
- 숫자는 정보의 중요도를 나타냄. 숫자가 작을수록 중요도가 높음.
- h1 태그는 가장 중요한 정보를 담으므로 하나의 문서에 한 번만 사용되는 걸 권장.
  - 네이버, 다음 같은 웹 사이트에선 회사의 이름이 가장 중요한 정보일 것이기 때문에 회사 이름을 h1 태그로 감쌀 것.
  -> ??? 개인 블로그일 경우엔?

<p>
: paragraph. 본문 내용을 담는다.
-> ??? 콘텐츠의 문단 별로 p 태그를 사용해야 되나? 아니면 내용 전체를 p 태그로 묶나?

<ul>
: unordered list. 순서가 없는 리스트를 만든다.
  - nav 구역의 메뉴 버튼을 만들 때 사용됨.
    - x축으로 정렬된 메뉴 버튼을 만들 땐 inline-block 속성을 사용하는 경우가 많음.

<a>
: anchor. 초월닻.

- 예) <a hrer="url">
: 클릭 시 해당 url로 이동.
  - 이동시킬 URL이 정해지지 않았으면 임시로 #를 넣어준다.

- 속성
  - target
    - 속성값
      1. _blank
      : 새탭에서 페이지 로드.
      2. _self
      : 현재 탭에서 페이지 로드. 기본값.

- 활용
  - 요소를 a 태그로 감싸기
  : 해당 요소에 '클릭 시 링크로 이동' 기능을 부여한다.
    - 예) 네이버 img를 a로 감싸면 해당 로고 이미지를 클릭했을 때 네이버 페이지로 이동함.

* 구조를 잡을 때 사용하는 태그

책의 구조
1. 목차
2. 본문
3. 부록

비유컨대
1. 목차
: 웹 사이트 최상단 네비게이션(메뉴) 영역.
2. 본문
: 주요 정보가 담겨진 영역.
3. 부록
: 웹 사이트의 최하단 회사와 관련된 정보가 담겨진 영역.

목차
- 예)

<header>
  <img>
  <nav>
    <ul>
      <li>
        <a>
        </a>
      </li>
    </ul>
  </nav>
</header>

<header>
: 웹사이트의 머리글. 사용자가 접속했을 때 가장 먼저 보여지는 영역.
- 일반적으로 웹 사이트 이름과 nav 영역을 포함하고 있음.

<nav>
: navigation. 메뉴 버튼을 담는 영역.
- 일반적으로 header 태그 안에 포함되어 있음.
- 일반적으로 x축 정렬된 메뉴 버튼은 inline-block 성격으로 만듦.
- 일반적으로 <ul> <li> <a> 와 함께 사용
- ul -> li -> a

본문
- 예)
<main role  = "main"> 본문 영역
<article> 정보 영역
<h></h> 제목 영역
</article>
</main>

<main>
: 메인 컨텐츠를 담는 영역.
- 일반적으로 article을 하위 영역으로 가짐.

- 속성
  - role="main"
  : 메인 태그의 역할을 설명한다.
    - 예) <main role="main"></main>
    - 익스플로러는 해당 태그의 으미를 이해하지 못하기 떄문.

<article>
: 주요 이미지나 텍스트 정보를 담는 영역.
- 웹 표준상 <article> 그룹 내부에는 각 영역을 대표하는 타이틀 정보가 반드시 필요함.
- 즉, h태그가 무조건 하나 이상은 필요함.

부록
- 예)
<footer>
<div>
<p>주소: ~~~</p>
<p>이메일: ~~~</p>
</div>
<div>
<p>사업자: ~~~</p>
</div>
</footer>

<footer>
: 웹 사이트의 가장 밑에 들어가는 정보를 담는 영역.
- 회사 주소, 이메일, 사업자 번호 등의 정보를 표시함.

<div>
: 임의의 공간을 만들 때 사용하는 태그.
- 거실, 화장실처럼 이미 적절한 의미를 갖는 태그가 있는 경우가 아닌 쇼파를 놓을 자리 티비를 놓을 자리 같은 성격의 애매한 공간들을 이 태그로 구획함.

* HTML 요소의 두 가지 성격

inline, block 성격의 특징

block
: 한 줄 전체를 요소의 영역으로 가진다.
- 줄바꿈 현상이 일어나 나열된 요소들이 y축 정렬 형태로 배치됨.
- 요소가 가진 영역의 사이즈를 자유롭게 조정할 수 있음.

inline
: 요소의 컨텐츠가 차지하는 공간만을 영역으로 가진다.
- 줄바꿈 현상이 일어나지 않아 나열된 요소들이 x축 정렬 형태로 배치됨.
- 요소가 가진 영역의 사이즈를 자유롭게 조정할 수 없음.
  - 상하 배치 작업이 불가능함.
  - 즉, margin-top/bottom 등의 CSS 속성이 적용되지 않음.

* CSS 주요 속성

CSS
: Cascading style sheet.
: HTML로 작성된 정보를 꾸며주거나 문서의 레이아웃을 설정하는 데 쓰인다.

구성
- 예)
선택자{
  속성: 속성값;
}
- 선택자
: 디자인을 적용할 HTML 영역.
- 속성
: 어떤 카테고리의 디자인을 적용할지 정의.
- 속성값
: 그 디자인 카테고리에서 구체적으로 뭘 적용할지 정의.

HTML 문서와의 연동 방법
1. inline
: HTML 태그 안에 원하는 스타일을 입력.
2. internal
: 스타일 태그를 헤드 태그 안에 입력.
3. external
: link 태그로 미리 만들어둔 CSS 파일을 불러와 적용.
- 예)
<link rel="stylesheet" href="css.css">
  - rel = ""
  : 연결된 문서가 어떠한 정보를 담고 있는지 설명.

- 장점
  - 각각의 언어로 작성된 코드를 따로 분리해 관리할 수 있음.
  - 상대적으로 가독성이 높고 유지 보수 측면에 강점이 있음.

CSS 선택자 종류

선택자
: CSS 언어가 HTML의 특정 요소를 선택하는 방법.
: 어떤 요소에 CSS를 적용할 것인가를 정함.
- 선택자 지정 방법
  - HTML 태그의 경우
  : 태그 이름으로 지정할 요소를 선택.
  - 클래스의 경우
  : '.'으로 지정할 요소를 선택.
  - 아이디의 경우
  : '#'으로 지정할 요소를 선택.

* 부모 자식 관계

부모 자식 관계
: 특정 태그가 다른 태그를 하위 태그로 갖는 형태.
  - 자식 태그는 부모 태그의 속성을 물려받음. 이를 '상속'이라고 함.

- 예)
<header>
<h1></h1>
<p></p>
</header>
  - header는 부모 태그가 된다.
  - h1, p는 자식 태그가 된다.
  - h1, p는 동급 관계로 형제 태그가 된다.

- 상속
  - 예)
  header{color: red;}
  h1{color: blue;}
  p{color: green;}
    - 부모 태그에 적용된 CSS를 자식 태그가 상속받기 때문에 처음엠 모두 빨간색으로 바뀜.
    - h1 태그와 p 태그는 형제 관계로서 각각에 CSS를 지정해도 서로에게 영향을 주지 않음.

* CSS의 우선 순위(캐스캐이딩)

CSS의 우선순위를 결정하는 세 가지 요소
- CSS의 우선 순위를 이용하면 원본 코드를 유지하며 CSS를 수정할 수 있음.

1. 적용되는 시점
: 시간적으로 나중에 덧씌워지는 CSS가 우선권을 가진다.
2. 디테일
: 더 구체적으로 작성된 선택자의 우선 순위가 높음(부모 태그까지 같이 적으면).
3. 선택자
: 선택자의 종류에 따라 우선 순위가 다름.
  - 타입 > 클래스 > 아이디
  - type < class < id 순으로 우선 순위가 높음

* CSS 주요 속성

width/heigt
: 폭/높이
- 고정값은 px
- 가변값은 %

font
: 텍스트의 모양

- 종류
  - font-size
  : 폰트의 사이즈
  - font-family
  : 폰트의 종류
    - 브라우저마다 지원하는 폰트가 다름.
    - 입력한 순서대로 우선 순위 적용.
    - serif, sans-serif, monospace는 마지막에 작성해준다. 모든 브라우저에서 지원하는 폰트기 때문에.
  - font-style:
  : 폰트의 스타일
  - font-weight
  : 폰트의 굵기
    - bold
    - 100~900 사이의 숫자를 입력할 수 있음.
      - 100단위로 올라간다.
      - 숫자가 클수록 굵어짐.

border
: 테두리

- 종류
  - boder-style: solid;
  : 테두리의 모양
    - solid
    : 실선
    - dotted
    : 점선
  - border-width: 1px;
  : 테두리 사이즈
    - 일반적으로 고정값을 사용함.
  - border-color: red;
  : 테두리 색깔
  - border: solid 1px red;
  : border 관련 속성의 축약형
    - 속성값은 띄어 쓰기로 구분. 순서는 상관없음.

background
: 특정 영역의 배경을 설정.

- 종류
  - background-color: yellow;
  : 배경 색 지정.
  - background-image: url(이미지 경로);
  : 배경 이미지 설정.
  - 괄호 안에 경로를 넣어야 함.
  - backgruound-repeat: no-repeat;
  : 배경 이미지를 반복.
    - repeat-x/y
    : x/y축으로 반복.
    - no-repaeat
    : 반복하지 않음.
  - background-position: left;
  : 공간 안에서 이미지의 좌표를 지정할 때
    - top, bottom center, left, right 등
  - background: yellow url no-repeat left;
  : background 관련 속성의 축약형
    - 순서는 상관없음.

2장. 웹사이트의 레이아웃

* 박스 모델

박스 모델
: 공간의 구성 요소를 박스 모양으로 나타낸 것.

- 구성 요소
  - margin
  : 테두리를 기준으로 바깥쪽 공간
  - border
  : 테두리
  - padding
  : 테두리를 기준으로 안쪽 공간
    - padding 값을 수정하면 전체 공간의 크기가 달라져 의도한 레이아웃으 나오지 않을 수 있으므로 유의.
    - HTML의 작은 단위부터 공간을 지정하기 떄문에.
    - HTML은 원래 문서를 작성하는 툴이었기 때문에 큰 공간의 레이아웃을 고려해야 할 필요가 없었으므로.
    - 뭔가 이상하다 싶으면 HTML의 근본적 사용처에 관련한 이유일 가능성이 큼.
  - content
  : 콘텐츠가 차지하는 공간

* block/inline 요소

block
- <header>, <div>, <p>, <h> 등.

inline
- <a>, <span> 등
- width, height, margin-top, margin-bottom 적용 불가.

* 마진 병합 현상

1. 동등한 관계에서(형제)
: 공통된 빈 공간 영역은 두 객체의 마진 값 중 큰 값으로 병합됨.
  - 근본적인 원인은 HTML이 원래 글쓰기 툴이었다는 점에 있음.

2. 상하 관계에서(부모자식)
: 자식 객체에 마진을 설정하면 자식 객체가 아닌 부모 객체에 적용된다.
- 해당 요소에 position: absolute; 를 적용하면 이런 상황을 피할 수 있음.

* 레이아웃에 영향을 미치는 속성

1. display
- 속성값을 block, inline, inline-block으로 줘서 요소의 성격을 바꿀 수 있음.

2. float
: 객체를 왼쪽/오른쪽부터 정렬시킨다.
- 부모 태그의 사이즈를 기준으로 왼쪽 끝 오른쪽 끝이 설정된다.
- 말 그대로 레이어를 공중에 띄우는 효과를 주기 때문에 여러 레이어가 겹쳐서 표시될 수 있음.
- 객체들이 겹쳐지지 않게 하기 위해서는 해당 속성을 각 객체에 연속적으로 부여해준다.
  - 예)
  .left{float: left;}
  .next{float: left;}
    - left 클래스 요소 옆에 next 클래스 요소가 나란히 정렬됨.
- 해당 속성이 적용된 영역의 크기보다 부모의 크기가 작다면?
  -> 줄바꿈이 되어 레이아웃이 틀어진다.
  -> 이런 특성을 활용해 의도한 레이아웃을 만들기도 한다.
- clear 속성
  : 이 속성이 적용된 태그부터 플롯 속성을 제거해준다.
  - 플롯 속성을 사용했다면 거의 항상 따라오는 속성.
  - 플롯을 마지막으로 사용한 지점 바로 다음에 나오는 태그에 clear 속성을 넣어준다.
    - clear: left/right/both
    : 앞선 특정 플롯 속성을 꺼준다.

브라우저와 공간 사이의 공백 제거하기
- *{margin: 0; padding: 0;}
  : 모든 요소의 마진과 패딩을 0으로 만들어준다.
- 요소가 기본적으로 가지고 있는 margin, padding 값이 존재하기 떄문에.
- 웹 사이트를 제작할 땐 이런 초기화 작업이 반드시 필요함.

* 별책 부록

인라인 콘텐츠의 높이를 중앙 정렬 시켜주는 방법
: 콘텐츠의 line-height 속성을 콘텐츠가 담겨 있는 공간의 높이와 같은 값으로 지정함.
  -> y축 중앙 정렬이 됨.

요소들 간의 공백 없애기
- 인라인 요소들 간에도 미세하게 공백이 들어간다. 이것은 인라인 요소의 태생적 속성이기 떄문에 *로 해결이 안 됨.
- 그래서 inline-block이 아니라 float를 사용해 이번 쇼핑몰 레이아웃을 잡아준 것.

img 요소 공백 없애기
- 이미지 태그는 태생적으로 이미지 밑에 미세한 공백을 갖고 있음. 그 공백을 제거해주기 위해 vertical-align: middle; 속성을 디폴트 값으로 넣어주는 게 좋음.

단축평가
: 표현식을 평가하는 도중에 평가 결과가 확정된 경우 나머지 평가 과정을 생략하는 것.

HTTP 프로토콜
: Hypertext Transfer Protocol. 상호 간에 정의한 규칙. 데이터를 주고 받기 위해 정해졌다.
: 내가 이렇게 보낼게 너는 이렇게 받아.
: 웹에서는 브라우저와 서버 간 데이터를 주고받기 위해 HTTP 프로토콜을 사용하고 있다.

- 특징
  - 상태가 없다. stateless.
  : 데이터를 주고 받기 위한 각각의 데이터 요청이 서로 독립적으로 관리된다.
  : 즉, 이전 데이터 요청과 다음 데이터 요청이 서로 관련이 없다.
    -> 서버는 별도의 추가 정보를 관리하지 않아도 된다.
    -> 다수의 요청 처리 및 서버의 부하를 줄일 수 있다.
    -> 서버와 클라이언트가 상태를 주고받을 때 스토리지와 쿠키가 필요하다.
