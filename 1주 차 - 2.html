수업 내용(에서 기억해둘 만한 것들)

1. 다른 네이티브들과 비교하여 HTML에 좀 이상한 부분이 있다면 HTML이 이상한 게 맞다.
: HTML은 애초에 글쓰기(정보를 담는)를 목적으로 탄생했기 때문이다. 웹에 정보를 표시하기 위한 언어. 꽤 오래된 언어다.
- 뭔가 이상하다. -> HTML은 애초에 글쓰기 툴이었는데 시간이 지나며 여러 역할을 하게 됐다. -> 그래서 글쓰기 툴로서의 미덕('현재 웹에서 HTML의 역할'이라는 관점에서 봤을 땐 비합리적인 요소)이 군데군데 남아 있게 된 것.
- 대충 이런 식으로 생각해보면 맞다. 세계사에서 '이런 일이 있었다고?'라는 생각이 들 때 '영국이 했나 보네'라고 생각하면 대충 맞는다는 말이 있다. 그거랑 비슷한 맥락인 듯하다.
- 네이티브라는 건 이런 것이다. 예를 들어 MS에서 윈도우 앱은 이 언어로 이렇게 저렇게 개발하시면 됩니다, 하고 가이드를 준다. 그럼 개발자들은 그 가이드에 맞춰 앱을 개발한다. 원래 글쓰기 툴이었는데 이것저것 기능을 붙여 흡사 태세우스의 배가 된 HTML과는 달리, 네이티브 언어는 애초에 '프로그램을 만든다'라는 목적을 가지고 기획된 언어인 것이다. 계획 도시처럼 비합리적인 부분이 없다('적다'가 맞을지도 모르겠다. 아직 잘 모른다.).
- HTML의 이상한(비합리적인) 부분의 예로는 '마진 병합 현상', '일관성 없는 태그 네이밍(어떤 건 단축어로, 어떤 건 풀네임으로)', '내부 컨텐츠 크기를 조정하면 바깥쪽 영역의 사이즈가 틀어지는 현상' 등이 있다.
  - 예전엔 사용되는 태그가 많지 않아서 최대한 줄여놔도 구분해내기 쉬었다.
  - 속도가 느렸기 때문에 데이터를 줄이려는 목적도 있었다.
- 요약컨대, 모든 게 완벽하고 설계자의 백년지대계적인 의도가 반영됐다고 생각하면 곤란한 것이다. 사실 이건 거의 모든 게 그렇다.
- 그러니까 지금 필요한 기술들이나 공부하자.

2. HTML 태그를 구분하는 방식
: 기능적(으로 구분), 의미적(으로 구분)

- 기능적
: 어떤 기능을 위해 사용되는 태그.

- 의미적
: 어떤 구역에 담긴 콘텐츠의 의미를 명확히 하기 위해 사용되는 태그.
  - 출력되는 화면에 영향을 주진 않지만 코드의 가독성을 높힌다.
  - 따라서 코드를 유지/관리하는 데 도움이 된다.
  - 검색 엔진이 효과적으로 파싱할 수 있게 된다. 영역의 이름에 따라 컴퓨터가 정보를 짐작할 수 있기 때문에.

CSS를 감각하는 방법
- 레이아웃을 결정하는 속성 그룹. display, position...
- 크기와 여백을 결정하는 속성 그룹. width, margin...
- 이런 식으로 구획하여 인식하는 걸 추천한다. 어떤 속성을 보면 아 이거는 레이아웃 작성 용이구나~ 아 이거는 크기를 조정하기 위한 거구나~ 아 이거는 꾸미기 위한 거구나~ 이렇게 의도적으로 구분해보는 습관을 들이자.
- display(레이아웃)와 position(상대적 위치) 속성은 정말 중요하다. 이것들만 알면 웬만한 건 다 작성해낼 수 있다.
- 나머지는 꾸미는 용도다. 디테일을 만드는 것. 이런 것들은 중요한 것 몇 개만 알아두고 필요할 때마다 검색해도 괜찮다.

CSS 선택자
- 선택자들 사이에 꺽쇠가 있으면
  : 부모 태그의 직계 자손에게만 CSS가 적용됨
  - ex) div > p
- 선택자들 사이에 띄어 쓰기만 있고 꺽쇠가 없으면
  : 그 밑에 있는 모든 자식들에게 CSS가 적용됨
  - ex) div p
- CSS는 공백이 중요함. 공백으로 의도를 구분하기 때문에.
- p .pizza
: p 태그의 자손 중 class 값이 pizza인 요소
- p.pizza
: p 태그 중 class 값이 pizza인 요소

%와 vw(h)
- %
  : 부모 요소의 특성을 기준으로 한 퍼센테이지.
- vw(h)
  : 다 상관없고 현재 보이는 웹 브라우저의 전체 사이즈를 기준으로 한 퍼센테이지.

백업 폰트
: 프로그래머가 지정한 폰트가 사용자의 시스템에 없을 수 있다. 하여 기본적으로 제공되는 폰트 몇 가지를 백업 폰트로 넣어둔다.
- serif
: 삐침 있는 폰트.
- sans-serif
: 삐침 없는 폰트.
- monospace
: 모든 글자의 폭이 동일한 폰트. 주로 프로그래밍에 쓰인다.

float
- float는 디스플레이 속성이 아닌데도 레이아웃에 영향을 미치는 속성이다.
- float의 근본적인 목적은 텍스트 중간에 이미지를 곁드리는 것이었다. 즉, 레이아웃을 작성하는 용도가 아니었다.
- 박스 모델이 기본이다. 그런데 float를 사용하면 사각형 기반의 디자인을 다각형으로 만들 수 있었다.
- float는 이제 잘 쓰이지 않는 기능이지만 지금까지 살아남아 있다. 다른 걸로는 구현해낼 수 없는 걸 만들어내기 때문에.
  - 그런 사람이 되자.
- 사실 이 과정에서 float를 가르치는 이유는,, 굉장힌 현실적인 이유이다. flex나 grid를 가르치는 데는 시간이 많이 걸리고 교육 과정의 특성상 배움의 결과가 눈에 바로바로 보여야 하기 때문에. 그래서 잘 쓰이지 않는 float를 가르치게 된 것.
  - 이런 말을 안 하는 게 좋았을 거라 생각한다. 다들 간절함을 품고 왔을 테니까. 학원이 현실적인 이유로 필요성이 낮은 기술을 가르쳐주는 거라고 실토해버린다면 그들이 광고했던 말했던 보장한다던 말들을 신뢰할 수 있을까? 이 부분은 굉장히 별로였다. 기만당한 느낌이었다.
- clear 속성으로 float 적용을 정지시킨다. '이제부터 줄바꿈'이라고 생각하면 쉽다.
